# UI/UX Pro Max - Design Intelligence

Comprehensive design guide for web and mobile applications.

## Quick Reference - Priority Rules

| Priority | Category | Impact | Key Checks |
|----------|----------|--------|------------|
| 1 | Accessibility | CRITICAL | Contrast 4.5:1, Alt text, Keyboard nav, Aria-labels |
| 2 | Touch & Interaction | CRITICAL | Min size 44×44px, 8px+ spacing, Loading feedback |
| 3 | Performance | HIGH | WebP/AVIF, Lazy loading, Reserve space (CLS < 0.1) |
| 4 | Style Selection | HIGH | Match product type, Consistency, SVG icons (no emoji) |
| 5 | Layout & Responsive | HIGH | Mobile-first breakpoints, Viewport meta, No horizontal scroll |
| 6 | Typography & Color | MEDIUM | Base 16px, Line-height 1.5, Semantic color tokens |
| 7 | Animation | MEDIUM | Duration 150–300ms, Motion conveys meaning |
| 8 | Forms & Feedback | MEDIUM | Visible labels, Error near field, Progressive disclosure |
| 9 | Navigation Patterns | HIGH | Predictable back, Bottom nav ≤5, Deep linking |
| 10 | Charts & Data | LOW | Legends, Tooltips, Accessible colors |

## Responsive Typography Scale (Industry Standard)

### Desktop (≥1024px)
- h1: 56-72px / line-height 1.1
- h2: 40-48px / line-height 1.15
- h3: 28-32px / line-height 1.2
- h4: 22-24px / line-height 1.3
- Body: 18-20px / line-height 1.5-1.6
- Small: 14-16px / line-height 1.5

### Tablet (768-1023px)
- h1: 44-56px / line-height 1.1
- h2: 32-40px / line-height 1.15
- h3: 24-28px / line-height 1.2
- Body: 16-18px / line-height 1.5

### Mobile (≤767px)
- h1: 36-44px / line-height 1.15
- h2: 28-32px / line-height 1.2
- h3: 22-24px / line-height 1.25
- Body: 16px (minimum) / line-height 1.5
- Small: 14px / line-height 1.4

## Spacing Scale (8pt grid)
4, 8, 12, 16, 24, 32, 48, 64, 80, 96, 128

## Standard Breakpoints
- Mobile: 375px
- Tablet: 768px
- Desktop: 1024px
- Wide: 1440px

## Section Padding Standards
- Desktop: 80-120px vertical, 24-156px horizontal
- Tablet: 64-80px vertical, 24-48px horizontal
- Mobile: 48-64px vertical, 16-20px horizontal

## Touch Targets
- Minimum: 44×44px (iOS) / 48×48dp (Android)
- Spacing between targets: ≥8px

## Form Best Practices
- Input height: ≥48px (mobile ≥44px)
- Visible labels (not placeholder-only)
- Error messages below field
- Required field indicators
- Loading/success/error states on submit
- Progressive disclosure for complex forms
